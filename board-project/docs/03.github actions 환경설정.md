# 03. GitHub Actions 환경설정 (Workflow, Secret)

이 문서는 GitHub Actions를 사용하여 코드가 `main` 브랜치에 푸시될 때 자동으로 빌드하고 시놀로지 서버에 배포하는 CI/CD 파이프라인을 구축하는 방법을 상세히 안내합니다.

## 1. GitHub Secrets 설정

GitHub Actions 워크플로우에서 사용할 민감한 정보(비밀번호, 토큰, 서버 정보 등)를 GitHub Secrets에 안전하게 저장해야 합니다.

1.  **GitHub 저장소 접속**: 웹 브라우저에서 프로젝트의 GitHub 저장소로 이동합니다.
2.  **Secrets 설정 메뉴 이동**: `Settings` 탭을 클릭하고, 왼쪽 메뉴에서 `Secrets and variables` > `Actions`를 선택합니다.
3.  **새로운 Secret 추가**: `New repository secret` 버튼을 클릭하여 다음 정보들을 각각 추가합니다.

    | Name | Value | 설명 |
    | :--- | :--- | :--- |
    | `DOCKER_HUB_USERNAME` | 본인의 Docker Hub 사용자 이름 | Docker Hub 로그인을 위한 사용자 이름 |
    | `DOCKER_HUB_ACCESS_TOKEN` | 발급받은 Docker Hub Access Token | Docker Hub 로그인을 위한 토큰 (비밀번호 대신 사용 권장) |
    | `SYNOLOGY_HOST` | 시놀로지 서버의 IP 주소 또는 도메인 | 배포 대상 서버 주소 |
    | `SYNOLOGY_USER` | 시놀로지 서버의 SSH 접속 계정명 | 배포 대상 서버 접속 계정 |
    | `SSH_PRIVATE_KEY` | `01.기본환경구성`에서 생성한 개인 키 내용 | 시놀로지 서버 SSH 접속을 위한 개인 키 |
    | `SYNOLOGY_SSH_PORT` | 시놀로지 서버의 SSH 포트 번호 | 기본값은 22, 변경 시 해당 포트 번호 입력 |
    | `PROJECT_PATH` | 시놀로지 서버 내 프로젝트 배포 경로 | 예: `/volume1/docker/board-project` |

    *   **Docker Hub Access Token 발급 방법**: Docker Hub 로그인 > 우측 상단 프로필 > `Account settings` > `Security` > `New Access Token` 생성.

## 2. GitHub Actions Workflow 파일 작성

GitHub Actions 워크플로우는 `.github/workflows` 디렉토리 내의 YAML 파일로 정의됩니다. 프로젝트 루트 디렉토리에 이미 `deploy.yml` 파일이 생성되어 있습니다.

### 2.1. `deploy.yml` 파일 구조 및 설명

`.github/workflows/deploy.yml` 파일의 주요 내용은 다음과 같습니다.

```yaml
name: CI/CD Pipeline

on:
  push:
    branches: [ main ] # main 브랜치에 push 이벤트 발생 시 실행

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v3 # 저장소 코드 체크아웃

      - name: Login to Docker Hub
        uses: docker/login-action@v2 # Docker Hub 로그인
        with:
          username: ${{ secrets.DOCKER_HUB_USERNAME }}
          password: ${{ secrets.DOCKER_HUB_ACCESS_TOKEN }}

      - name: Build and Push Docker image
        uses: docker/build-push-action@v4 # Docker 이미지 빌드 및 푸시
        with:
          context: .
          push: true
          tags: ${{ secrets.DOCKER_HUB_USERNAME }}/board-app:latest

  deploy:
    needs: build-and-push # build-and-push 작업 완료 후 실행
    runs-on: ubuntu-latest
    steps:
      - name: SSH Deploy to Synology
        uses: appleboy/ssh-action@master # SSH를 통해 시놀로지 서버 접속 및 배포 스크립트 실행
        with:
          host: ${{ secrets.SYNOLOGY_HOST }}
          username: ${{ secrets.SYNOLOGY_USER }}
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          port: ${{ secrets.SYNOLOGY_SSH_PORT }}
          script: |
            cd ${{ secrets.PROJECT_PATH }}
            docker-compose pull # 최신 이미지 다운로드
            docker-compose up -d # 컨테이너 백그라운드 실행
```

### 2.2. Workflow 실행 확인

1.  로컬에서 코드를 수정하고 커밋한 후 `main` 브랜치로 푸시합니다.
2.  GitHub 저장소의 `Actions` 탭으로 이동합니다.
3.  `CI/CD Pipeline` 워크플로우가 실행 중인지 확인합니다.
4.  각 단계(build-and-push, deploy)가 성공적으로 완료되었는지 로그를 통해 확인합니다.

이로써 GitHub Actions 환경 설정이 완료되었습니다. 다음 문서에서는 시놀로지 Docker Compose 환경 설정에 대해 안내합니다.
