# 02. GitHub 프로젝트 설정 및 기준 브랜치 설정

이 문서는 GitHub에 프로젝트를 생성하고, Gitflow 전략을 기반으로 기준 브랜치를 설정하는 방법을 안내합니다. 이는 협업 및 버전 관리를 효율적으로 수행하기 위한 필수적인 단계입니다.

## 1. GitHub 저장소 생성

1.  **GitHub 접속**: 웹 브라우저를 열고 [GitHub](https://github.com/)에 접속하여 로그인합니다.

2.  **새 저장소 생성**: GitHub 페이지 우측 상단의 `+` 버튼을 클릭한 후 `New repository`를 선택합니다.

3.  **저장소 정보 입력**: 다음 정보를 입력하여 저장소를 생성합니다.
    *   **Repository name**: `board-project` (또는 원하는 프로젝트 이름)
    *   **Description (Optional)**: 프로젝트에 대한 간략한 설명
    *   **Public/Private**: 프로젝트의 공개 여부를 선택합니다. (일반적으로 Private으로 시작하여 필요시 Public으로 전환)
    *   **Initialize this repository with**: `Add a README file`을 선택하여 초기 README 파일을 생성합니다.
    *   `.gitignore`: `Java` 템플릿을 추가하여 Java 프로젝트에 맞는 불필요한 파일들이 Git에 포함되지 않도록 합니다.
    *   **License**: 원하는 라이선스를 선택합니다 (선택 사항).

4.  **저장소 생성 완료**: `Create repository` 버튼을 클릭하여 저장소 생성을 완료합니다.

## 2. 로컬 프로젝트를 GitHub 저장소에 연결

생성된 GitHub 저장소에 로컬 프로젝트 코드를 업로드합니다.

1.  **로컬 프로젝트 초기화**: 로컬 프로젝트 루트 디렉토리(`/home/ubuntu/board-project`)에서 다음 명령어를 실행하여 Git 저장소를 초기화합니다.

    ```bash
    git init
    ```

2.  **원격 저장소 추가**: GitHub에서 생성한 저장소의 URL을 복사하여 원격 저장소로 추가합니다.

    ```bash
    git remote add origin https://github.com/your-username/board-project.git
    ```

3.  **파일 추가 및 커밋**: 프로젝트의 모든 파일을 스테이징 영역에 추가하고 커밋합니다.

    ```bash
    git add .
    git commit -m "Initial commit: Project setup"
    ```

4.  **GitHub에 푸시**: `main` 브랜치로 GitHub 저장소에 코드를 푸시합니다.

    ```bash
    git branch -M main
    git push -u origin main
    ```

## 3. 기준 브랜치 설정 (Gitflow 전략)

Gitflow 전략은 안정적인 릴리스 관리를 위해 `main`, `develop` 브랜치를 중심으로 운영됩니다.

*   **`main` 브랜치**: 항상 배포 가능한 상태의 코드를 유지합니다. (운영 환경 배포용)
*   **`develop` 브랜치**: 다음 릴리스를 위한 개발이 진행되는 브랜치입니다. (개발 환경 배포용)

### 3.1. `develop` 브랜치 생성 및 푸시

`main` 브랜치에서 `develop` 브랜치를 생성하고 GitHub에 푸시합니다.

```bash
git checkout -b develop
git push -u origin develop
```

### 3.2. GitHub에서 브랜치 보호 규칙 설정

안정적인 코드 관리를 위해 `main` 및 `develop` 브랜치에 보호 규칙을 설정합니다.

1.  **GitHub 저장소 설정 접속**: GitHub 저장소 > `Settings` > `Branches`로 이동합니다.

2.  **브랜치 보호 규칙 추가**: `Add branch protection rule` 버튼을 클릭합니다.

3.  **`main` 브랜치 보호 규칙 설정**: `Branch name pattern`에 `main`을 입력하고 다음 옵션들을 설정합니다.
    *   `Require a pull request before merging`: `main` 브랜치에 직접 푸시하는 것을 방지하고, Pull Request를 통해서만 병합되도록 합니다.
    *   `Require approvals`: 코드 리뷰를 필수로 지정합니다.
    *   `Require status checks to pass before merging`: CI/CD 테스트 통과를 필수로 지정합니다.
    *   `Include administrators`: 관리자에게도 규칙을 적용합니다.

4.  **`develop` 브랜치 보호 규칙 설정**: `develop` 브랜치에 대해서도 위와 동일하게 보호 규칙을 설정합니다.

이로써 GitHub 프로젝트 설정 및 기준 브랜치 설정이 완료되었습니다. 다음 문서에서는 GitHub Actions 환경 설정에 대해 안내합니다.
