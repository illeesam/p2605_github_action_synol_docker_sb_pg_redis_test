# 04. 시놀로지 Docker Compose (Container Manager) 환경설정

이 문서는 시놀로지 NAS의 Container Manager (이전 Docker)를 사용하여 `docker-compose.yml` 파일을 기반으로 Spring Boot 및 JSP 애플리케이션을 배포하는 방법을 상세하게 안내합니다.

## 1. Container Manager 설치 및 설정

시놀로지 NAS에 Container Manager가 설치되어 있지 않다면, 먼저 설치해야 합니다.

1.  **패키지 센터 접속**: 시놀로지 DSM에 로그인한 후 `패키지 센터`를 실행합니다.
2.  **Container Manager 설치**: 검색창에 `Container Manager`를 검색하여 설치합니다. (이미 설치되어 있다면 이 단계는 건너뜁니다.)

## 2. 프로젝트 디렉토리 생성 및 `.env` 파일 업로드

GitHub Actions에서 SSH를 통해 접근할 프로젝트 디렉토리를 시놀로지 NAS에 생성하고, 환경 변수 파일인 `.env`를 업로드합니다.

1.  **공유 폴더 생성 (선택 사항)**: Docker 컨테이너에서 사용할 데이터를 저장할 공유 폴더를 생성하는 것이 좋습니다. 예를 들어, `docker`라는 공유 폴더를 생성하고 그 안에 `board-project` 폴더를 생성합니다. (`/volume1/docker/board-project`)

2.  **`.env` 파일 생성 및 업로드**: `01.기본환경구성` 문서에서 생성한 `.env` 파일을 시놀로지 NAS의 프로젝트 디렉토리 (`/volume1/docker/board-project`)에 업로드합니다. 이 파일에는 `DB_PASSWORD`, `DOCKER_HUB_USERNAME`, `PROJECT_PATH` 등의 환경 변수가 포함되어야 합니다.

    ```dotenv
    DB_PASSWORD=your_postgresql_password
    DOCKER_HUB_USERNAME=your_docker_hub_username
    PROJECT_PATH=/volume1/docker/board-project
    ```

3.  **`docker-compose.yml` 파일 업로드**: 프로젝트 루트에 있는 `docker-compose.yml` 파일도 동일한 디렉토리 (`/volume1/docker/board-project`)에 업로드합니다.

## 3. Container Manager에서 프로젝트 생성 및 실행

시놀로지 Container Manager는 `docker-compose.yml` 파일을 사용하여 애플리케이션 스택을 쉽게 배포할 수 있는 기능을 제공합니다.

1.  **Container Manager 실행**: 시놀로지 DSM에서 `Container Manager`를 실행합니다.

2.  **프로젝트 탭 이동**: 왼쪽 메뉴에서 `프로젝트` 탭을 클릭합니다.

3.  **생성 버튼 클릭**: `생성` 버튼을 클릭하여 새로운 프로젝트를 생성합니다.

4.  **프로젝트 설정**: 다음 정보를 입력합니다.
    *   **프로젝트 이름**: `board-project` (원하는 이름으로 설정)
    *   **경로**: `docker-compose.yml` 파일이 위치한 시놀로지 NAS의 경로를 지정합니다. (예: `/volume1/docker/board-project`)
    *   **환경 변수**: `01.기본환경구성`에서 생성한 `.env` 파일의 경로를 지정합니다. (예: `/volume1/docker/board-project/.env`)

5.  **다음 클릭**: 설정을 확인하고 `다음` 버튼을 클릭합니다.

6.  **웹훅 설정 (선택 사항)**: GitHub Actions와 연동하여 자동 배포를 구성할 경우 웹훅을 설정할 수 있습니다. `웹훅 활성화`를 체크하고, 생성된 웹훅 URL을 복사합니다. 이 URL은 GitHub Actions의 `deploy` 단계에서 `script`에 추가하여 배포 완료 후 Container Manager에 알리는 데 사용될 수 있습니다. (현재 `deploy.yml`에서는 SSH를 통해 직접 `docker-compose` 명령어를 실행하므로 웹훅은 필수는 아닙니다.)

7.  **설정 요약 확인**: 프로젝트 설정 요약을 확인하고 `완료` 버튼을 클릭합니다.

8.  **프로젝트 실행 확인**: 프로젝트가 생성되면 자동으로 `docker-compose.yml` 파일에 정의된 서비스(db, redis, app)들이 빌드되고 실행됩니다. `프로젝트` 탭에서 `board-project`의 상태가 `실행 중`인지 확인합니다.

## 4. 컨테이너 및 로그 확인

배포된 컨테이너의 상태와 로그를 확인하여 정상적으로 작동하는지 점검합니다.

1.  **컨테이너 탭 이동**: Container Manager의 왼쪽 메뉴에서 `컨테이너` 탭을 클릭합니다.
2.  **컨테이너 상태 확인**: `board-db`, `board-redis`, `board-app` 컨테이너가 모두 `실행 중`인지 확인합니다.
3.  **로그 확인**: 각 컨테이너를 선택한 후 `로그` 탭을 클릭하여 컨테이너의 실행 로그를 확인합니다. 오류 메시지가 없는지 확인합니다.

이로써 시놀로지 Docker Compose 환경 설정이 완료되었습니다. 다음 문서에서는 로컬에서 시놀로지 직접 연결 테스트 방법에 대해 안내합니다.
