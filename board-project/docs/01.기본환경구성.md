# 01. 기본 환경 구성

이 문서는 Spring Boot 및 JSP 기반 게시판 프로젝트를 위한 개발 및 배포 환경을 설정하는 방법을 안내합니다. SSH 키 생성 및 시놀로지 환경 변수 설정에 중점을 둡니다.

## 1. SSH 키 생성 및 설정

GitHub Actions에서 시놀로지 서버로 안전하게 접속하기 위해 SSH 키를 생성하고 설정해야 합니다.

### 1.1. 로컬 환경에서 SSH 키 생성

터미널 또는 Git Bash를 열고 다음 명령어를 실행하여 SSH 키 쌍을 생성합니다.

```bash
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

*   `-t rsa`: RSA 알고리즘을 사용합니다.
*   `-b 4096`: 4096비트 길이의 키를 생성합니다 (보안 권장).
*   `-C "your_email@example.com"`: 키에 대한 주석을 추가합니다. 본인의 이메일 주소를 사용하세요.

명령어를 실행하면 키를 저장할 위치와 비밀번호를 묻습니다. 기본 경로(`~/.ssh/id_rsa`)를 사용하고, 비밀번호는 설정하지 않거나 기억하기 쉬운 비밀번호를 설정할 수 있습니다. (GitHub Actions에서는 비밀번호 없는 키가 더 편리합니다.)

생성된 파일은 두 가지입니다:
*   `id_rsa`: **개인 키 (Private Key)** - 외부에 노출되지 않도록 주의해야 합니다.
*   `id_rsa.pub`: **공개 키 (Public Key)** - 서버에 등록하여 접속을 허용하는 데 사용됩니다.

### 1.2. 시놀로지 서버에 공개 키 등록

생성된 `id_rsa.pub` 파일의 내용을 시놀로지 서버에 등록해야 합니다.

1.  **공개 키 내용 복사**: `cat ~/.ssh/id_rsa.pub` 명령어를 사용하여 공개 키 내용을 복사합니다.

2.  **시놀로지 SSH 접속**: Mobaxterm 또는 터미널을 사용하여 시놀로지 서버에 SSH로 접속합니다.

    ```bash
    ssh your_synology_user@your_synology_ip -p your_ssh_port
    ```

3.  **`authorized_keys` 파일 생성 또는 수정**: 시놀로지 서버에 접속한 후, 다음 명령어를 실행하여 `.ssh` 디렉토리를 생성하고 `authorized_keys` 파일에 공개 키를 추가합니다.

    ```bash
    mkdir -p ~/.ssh
    chmod 700 ~/.ssh
    echo "[복사한 공개 키 내용]" >> ~/.ssh/authorized_keys
    chmod 600 ~/.ssh/authorized_keys
    ```

    `[복사한 공개 키 내용]` 부분에 `id_rsa.pub` 파일의 전체 내용을 붙여넣습니다.

4.  **SSH 설정 확인**: 시놀로지 DSM에서 제어판 > 터미널 및 SNMP > SSH 활성화 및 포트 확인 (기본 22번).

### 1.3. GitHub Secrets에 개인 키 등록

GitHub Actions에서 시놀로지 서버에 접속할 때 사용할 개인 키(`id_rsa`)를 GitHub 저장소의 Secrets에 등록합니다. **개인 키는 절대 공개 저장소에 직접 올리면 안 됩니다.**

1.  **개인 키 내용 복사**: `cat ~/.ssh/id_rsa` 명령어를 사용하여 개인 키 내용을 복사합니다.

2.  **GitHub 저장소 Secrets 설정**: GitHub 저장소 > Settings > Security > Secrets and variables > Actions로 이동합니다.

3.  **새 저장소 비밀 추가**: `New repository secret` 버튼을 클릭하고 다음 정보를 입력합니다.
    *   **Name**: `SSH_PRIVATE_KEY`
    *   **Secret**: 복사한 `id_rsa` 개인 키 내용을 붙여넣습니다.

## 2. 시놀로지 환경 변수 설정 (`.env` 파일)

시놀로지 서버에서 Docker Compose를 실행할 때 필요한 민감한 정보(예: 데이터베이스 비밀번호)는 `.env` 파일에 저장하여 관리하는 것이 좋습니다. 이 파일은 `.gitignore`에 추가하여 버전 관리에서 제외해야 합니다.

1.  **프로젝트 루트 디렉토리 생성**: 시놀로지 서버에서 프로젝트를 배포할 디렉토리를 생성합니다. 예를 들어 `/volume1/docker/board-project`.

2.  **`.env` 파일 생성**: 해당 디렉토리 내에 `.env` 파일을 생성하고 다음 내용을 추가합니다.

    ```dotenv
    DB_PASSWORD=your_postgresql_password
    DOCKER_HUB_USERNAME=your_docker_hub_username
    PROJECT_PATH=/volume1/docker/board-project
    ```

    *   `DB_PASSWORD`: PostgreSQL 데이터베이스의 비밀번호를 입력합니다.
    *   `DOCKER_HUB_USERNAME`: Docker Hub 사용자 이름을 입력합니다.
    *   `PROJECT_PATH`: 시놀로지 서버에서 프로젝트가 배포될 경로를 입력합니다.

3.  **`.env` 파일 권한 설정**: `.env` 파일의 권한을 적절히 설정하여 민감한 정보가 노출되지 않도록 합니다.

    ```bash
    chmod 600 .env
    ```

이로써 기본 환경 구성이 완료되었습니다. 다음 문서에서는 GitHub 프로젝트 설정에 대해 안내합니다.
