# 00. 프로젝트 개요

본 문서는 Spring Boot 기반의 백엔드와 JSP 프론트엔드로 구성된 게시판 프로젝트의 전반적인 구조, 개발 환경, 배포 전략 및 관련 매뉴얼에 대한 개요를 제공합니다.

## 1. 프로젝트 목표

사용자는 본 프로젝트를 통해 다음과 같은 목표를 달성할 수 있습니다.

*   **게시판 기능 구현**: 간단한 게시판 기능을 JSP를 통해 제공합니다.
*   **백엔드 API 개발**: Spring Boot를 활용하여 RESTful API를 구축하고, PostgreSQL 데이터베이스 및 Redis와 연동하여 데이터 관리를 수행합니다.
*   **프론트엔드 UI 개발**: JSP를 사용하여 웹 인터페이스를 개발하고, 백엔드 API와 연동하여 동적인 콘텐츠를 제공합니다.
*   **CI/CD 파이프라인 구축**: GitHub Actions를 활용하여 코드 변경 시 자동으로 빌드, Docker 이미지 생성 및 Docker Hub에 푸시하는 CI (Continuous Integration) 환경을 구축합니다.
*   **시놀로지 NAS 배포 자동화**: Docker Compose를 이용하여 시놀로지 NAS에 백엔드, 데이터베이스, Redis 컨테이너를 효율적으로 배포하고 관리하는 CD (Continuous Deployment) 환경을 구축합니다.
*   **상세 매뉴얼 제공**: 개발 및 배포 과정 전반에 걸쳐 필요한 상세 매뉴얼을 제공하여 사용자가 쉽게 프로젝트를 이해하고 활용할 수 있도록 돕습니다.

## 2. 주요 기술 스택

본 프로젝트는 다음과 같은 주요 기술 스택을 활용하여 개발되었습니다.

| 구분 | 기술 스택 | 설명 |
| :--- | :--- | :--- |
| **백엔드** | Spring Boot 3.x | Java 기반의 웹 애플리케이션 프레임워크 |
| | Java 17 | 프로그래밍 언어 |
| | Gradle | 빌드 자동화 도구 |
| | PostgreSQL 15.x | 관계형 데이터베이스 |
| | Redis | 인메모리 데이터 구조 스토어 |
| **프론트엔드** | JSP (JavaServer Pages) | 서버 사이드 웹 페이지 기술 |
| **CI/CD & 배포** | GitHub Actions | 코드 변경 시 자동 빌드 및 배포 워크플로우 |
| | Docker | 컨테이너 기반 가상화 플랫폼 |
| | Docker Compose | 다중 컨테이너 Docker 애플리케이션 정의 및 실행 도구 |
| | Synology Container Manager | 시놀로지 NAS에서 Docker 컨테이너 관리를 위한 도구 |
| **개발 도구** | VS Code, IntelliJ, Mobaxterm | 개발 및 서버 접속을 위한 통합 개발 환경 및 터미널 에뮬레이터 |

## 3. 프로젝트 구조

프로젝트는 크게 `src/main/java`, `src/main/resources`, `src/main/webapp/WEB-INF/jsp`, `docs`, `.github/workflows` 디렉토리로 구성됩니다.

```
board-project/
├── src/                      # 소스 코드 디렉토리
│   ├── main/
│   │   ├── java/             # Java 소스 코드
│   │   │   └── com/example/board/
│   │   │       ├── BoardApplication.java
│   │   │       └── controller/MainController.java
│   │   ├── resources/        # 리소스 파일 (application.yml)
│   │   │   └── application.yml
│   │   └── webapp/           # 웹 애플리케이션 리소스
│   │       └── WEB-INF/
│   │           └── jsp/      # JSP 파일
│   │               ├── index.jsp
│   │               ├── echo.jsp
│   │               ├── db-test.jsp
│   │               ├── postgresql-test.jsp
│   │               └── mysql-test.jsp
├── docs/                     # 상세 매뉴얼 문서
│   ├── 00.프로젝트_개요.md
│   ├── 01.기본환경구성.md
│   ├── 02.github 프로젝트설정 및 기준브랜치 설정.md
│   ├── 03.github actions 환경설정.md
│   ├── 04.시놀로지 docker compose (container manager)환경설정.md
│   ├── 05.로컬에서 시놀로지 직접연결 테스트, 점검, 트러블슈팅.md
│   └── 06.github에서 시놀로지 배포 테스트, 점검, 트러블슈팅.md
├── .github/workflows/        # GitHub Actions 워크플로우 정의
│   └── deploy.yml
├── build.gradle              # Gradle 빌드 파일
├── Dockerfile                # Docker 이미지 빌드 파일
└── docker-compose.yml        # Docker Compose 설정 파일
```

## 4. 매뉴얼 목록

본 프로젝트와 함께 제공되는 상세 매뉴얼은 다음과 같습니다.

1.  **00. 프로젝트 개요**: 프로젝트의 전반적인 목표, 기술 스택, 구조에 대한 설명.
2.  **01. 기본 환경 구성**: SSH 키 생성 및 설정, 시놀로지 환경 변수(`.env`) 설정 방법.
3.  **02. GitHub 프로젝트 설정 및 기준 브랜치 설정**: GitHub 저장소 생성, 로컬 프로젝트 연결, Gitflow 전략 기반 브랜치 설정 방법.
4.  **03. GitHub Actions 환경설정 (Workflow, Secret)**: GitHub Secrets 설정 및 CI/CD 워크플로우 (`deploy.yml`) 구성 방법.
5.  **04. 시놀로지 Docker Compose (Container Manager) 환경설정**: 시놀로지 NAS에 Docker Compose를 이용한 애플리케이션 배포 방법.
6.  **05. 로컬에서 시놀로지 직접 연결 테스트, 점검, 트러블슈팅**: 로컬 환경에서 시놀로지 배포 서비스 테스트 및 문제 해결 방법.
7.  **06. GitHub에서 시놀로지 배포 테스트, 점검, 트러블슈팅**: GitHub Actions를 통한 자동 배포 테스트 및 문제 해결 방법.

이 매뉴얼들을 통해 사용자는 프로젝트의 개발부터 배포, 그리고 문제 해결까지 전 과정을 이해하고 직접 수행할 수 있습니다.
